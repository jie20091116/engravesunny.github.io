React路由

路由的严格匹配与模糊匹配

  • 默认使用模糊匹配（输入路径必须包含要匹配的路径，且顺序一致
  • 开启严格模式：<Route exact path='/about' component={About}/>
  • 严格匹配开启可能会导致无法继续匹配二级路由

Redirect的使用

  • 一般写在所有路由注册的最下方，当所有路由无法匹配时，跳转到Redirect指定的路由

<Switch>
  <Route path="/about" component="&#123;About&#125;" />
  <Route path="/home" component="&#123;Home&#125;" />
  <Redirect to="/about" />
</Switch>

嵌套路由

  • 注册子路由上需要写父路由的path
  • 路由的匹配是按照注册路由的顺序进行的
<!-- 父组件 -->
<MyNavLink to="/about">About</MyNavLink>
<MyNavLink to="/home">Home</MyNavLink>

<Switch>
  <Route path="/about" component="&#123;About&#125;" />
  <Route path="/home" component="&#123;Home&#125;" />
  <Redirect to="/about" />
</Switch>

<!-- 子组件 -->
<ul className="nav nav-tabs">
  <li>
    <MyNavLink to="/home/news">News</MyNavLink>
  </li>
  <li>
    <MyNavLink to="/home/message">Message</MyNavLink>
  </li>
</ul>

<Switch>
  <Route path="/home/news" component="&#123;News&#125;" />
  <Route path="/home/message" component="&#123;Message&#125;" />
  <Redirect to="/home/news" />
</Switch>

路由传参

三种方式：params,search,state
三种方式对比：

  • state:在当前页面刷新可以保留参数，但在新页面刷新不能保留。前两种方式因为是在url地址上保存参数，因此地址都能保留
  • params和search参数都会变成字符串
<!-- 路由链接 -->
<Link to='/home/message/detail/Bruce/21'>params</Link>
<Link to=&#123;`/home/message/detail/$&#123;item.name&#125;/$&#123;item.age&#125;`&#125;>&#123;item.name&#125;</Link>

<Link to='/home/message/detail/?name=Bruce&age=21'>search</Link>
<Link to=&#123;`/home/message/detail/?id=$&#123;item.name&#125;&title=$&#123;item.age&#125;`&#125;>&#123;item.name&#125;</Link>

<Link to=&#123;&#123;pathname: '/home/message/detail', state: &#123;name: 'Bruce', age: 21&#125;&#125;&#125;>state</Link>

<!-- 注册路由 -->
<Route path='/home/message/detail/:name/:age' component=&#123;Detail&#125; />
<!-- search 和 state 按正常注册即可 -->
<Route path='/home/message/detail' component=&#123;Detail&#125; />

import qs from 'querystring'

const &#123; name, age &#125; = qs.parse(search.slice(1))

编程式导航

编程式导航是使用路由组件this.props.history提供的API进行路由跳转

this.props.history.push(path,state)
this.props.history.replace(path,state)
this.props.history.goForward()
this.props.history.gopBack()
this.props.history.go(2)参数为负数时是后退

//编程式导航传参数
this.props.history.push(`/home/message/detail/$&#123;id&#125;/$&#123;title&#125;`)//params传参
this.props.history.push(`/home/message/detail?id=&#123;id&#125;&title=&#123;title&#125;`)//search传参
this.props.history.push(`/home/message/detail`,&#123;id,title&#125;)//state传参

BrowserRouter和HashRouter

底层原理不一样

  • BrowserRouter使用的是H5的history API，不兼容IE9及以下版本
  • HashRouter使用的是URL的哈希值

路径表现不一样

  • BrowserRouter的路径中没有#，如：localhost:3000/demo/test
  • HashRouter的路径包含#，如：localhost:3000/demo/test

刷新会对路由state参数的影响

  • BrowserRouter没有影响，因为state保存在history对象中
  • HashRouter刷新后会导致路由state参数的丢失

备注:HashRouter可以解决一些路径错误相关的问题

Ant Design 配置按需引入和自定义主题

以下配置是 3.x 版本，4.x 版本见官网

1.安装依赖：

npm install react-app-rewired customize-cra babel-plugin-import less less-loader

2.修改 package.json

  "start": "react-app-rewired start",
  "build": "react-app-rewired build",
  "test": "react-app-rewired test",
  "eject": "react-scripts eject"
&#125;

3.根目录下创建 config-overrides.js

  &#125;),

  &#125;)

备注：不用在组件里引入样式，import ‘antd/dist/antd.css’ 删掉
